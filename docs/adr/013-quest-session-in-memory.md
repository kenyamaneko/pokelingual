# ADR-013: クエストセッションのインメモリ管理

## ステータス

Accepted

## 結論

数分で完了する短いクエストのレイテンシを抑えるため、クエストセッションを `sync.Map` でインメモリ管理する。Firestore 往復を省いて高速に動作し、Cloud Run が単一インスタンスのためセッション衝突も起きない。

## 背景・課題

クエストは new → score → guess-name → capture の 4 段階フローで、各ステップ間でポケモン情報やスコアを保持する必要がある。クエストは短時間で完了し、永続化の必要性は低い。

## 制約

Cloud Run をスケールアウトすると、同一ユーザーが異なるインスタンスに振られたときにセッションが見つからない。現状のユーザー数（1-2 人）では問題にならず、スケールが必要になった時点で Redis 等への移行を検討する。

## 不採用案

- **Firestore に保存**：永続的だが、ステップごとの往復でレイテンシが高い。数分で完了する短いフローに永続化は過剰。
