# ADR-010: 種族値ベースの捕獲率計算式

## ステータス

採用済み

## コンテキスト

従来の捕獲率は `score × nameMultiplier × guessPenalty` という単純な線形計算だった。
全ポケモンが同じ条件で捕まるため、強いポケモン（高種族値）を捕まえた達成感がなく、ゲーム性が物足りなかった。

ポケモン本家ゲームでは種族値が高いポケモンほど捕獲が難しい仕組みがあり、同様の体験を翻訳ゲームでも実現したかった。

## 決定

### 捕獲率の計算式

シグモイド関数ベースの捕獲率計算を採用:

```
X = BST / 100,  S = score / 100
logit = 3.12 - 0.34X - 0.17X² + 12.78S - 4.83XS + 0.58X²S
captureRate = clamp(sigmoid(logit) × ballMultiplier, 0, 1)
```

感覚値（モンスターボール使用時）:

| BST \ Score | 30 | 50 | 70 | 90 |
|---|---|---|---|---|
| 300（弱い） | ~84% | ~91% | ~95% | ~98% |
| 400 | ~47% | ~60% | ~72% | ~82% |
| 500 | ~13% | ~22% | ~35% | ~50% |
| 600（強い） | ~3% | ~6% | ~15% | ~30% |

### ボール種類（名前当て結果で決定）

従来の `nameMultiplier`（1.5 / 1.0 / 0.5）と `wrongGuessPenalty` を廃止し、
ボール種類に一本化:

| 名前当て結果 | ボール | 倍率 |
|---|---|---|
| 英語名 正解（完全 or ファジー） | ハイパーボール（ultra） | 3.0x |
| 日本語名 正解 | スーパーボール（great） | 2.0x |
| スキップ・3回失敗 | モンスターボール（poke） | 1.0x |

### ポケモン名伏せ字

出題時の説明文にポケモン名が含まれる場合、名前推測のヒントになるのを防ぐため、
自然な代名詞に置換する:

- 英語: 文脈に応じて "this Pokémon"（単数）/ "of these Pokémon"（複数）
  - 複数形の判定: 直前の単語が several / many / few / multiple 等 → 複数形
  - 文頭 → 先頭大文字 "This Pokémon"
- 日本語: "この ポケモン"

AI スコアリングには元の（マスクなし）テキストを使用し、採点精度に影響しない。
捕獲結果とコレクションでは元の名前を表示する。

## 結果

- 強いポケモンほど捕まえにくく、ゲーム性が向上
- ボール種類による視覚的フィードバック（既存のフロントエンド UI と整合）
- 名前伏せ字により、名前推測がより挑戦的に
- `QuestSession` から `NameMultiplier` / `WrongGuesses` を削除し、`BaseStatTotal` / `BallType` に置き換え
- PokeAPI `/api/v2/pokemon/{id}` の `stats` フィールドから BST を自動計算
