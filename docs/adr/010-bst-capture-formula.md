# ADR-010: 種族値ベースの捕獲率計算式

## ステータス

Accepted

## 結論

強いポケモンほど捕まえにくくしてゲーム性を高めるため、捕獲率を種族値（BST）ベースのシグモイド関数で計算する。名前当ての結果でボール種類（倍率）を決め、出題文中のポケモン名は伏せ字にする。強敵の捕獲に達成感が生まれ、名前推測がより挑戦的になる。

## 背景・課題

従来の捕獲率は `score × nameMultiplier × guessPenalty` の単純な線形計算で、全ポケモンが同じ条件で捕まった。強いポケモン（高種族値）を捕まえた達成感がなく、ゲーム性が物足りなかった。ポケモン本家では種族値が高いほど捕獲が難しく、同様の体験を翻訳ゲームでも実現したかった。

## 詳細

### 捕獲率の計算式

```
X = BST / 100,  S = score / 100
logit = 3.12 - 0.34X - 0.17X² + 12.78S - 4.83XS + 0.58X²S
captureRate = clamp(sigmoid(logit) × ballMultiplier, 0, 1)
```

感覚値（モンスターボール使用時）：

| BST \ Score | 30 | 50 | 70 | 90 |
|---|---|---|---|---|
| 300（弱い） | ~84% | ~91% | ~95% | ~98% |
| 400 | ~47% | ~60% | ~72% | ~82% |
| 500 | ~13% | ~22% | ~35% | ~50% |
| 600（強い） | ~3% | ~6% | ~15% | ~30% |

### ボール種類（名前当て結果で決定）

従来の `nameMultiplier`（1.5 / 1.0 / 0.5）と `wrongGuessPenalty` を廃止し、ボール種類に一本化する。

| 名前当て結果 | ボール | 倍率 |
|---|---|---|
| 英語名 正解（完全 or ファジー） | ハイパーボール（ultra） | 3.0x |
| 日本語名 正解 | スーパーボール（great） | 2.0x |
| スキップ・3回失敗 | モンスターボール（poke） | 1.0x |

### ポケモン名伏せ字

出題時の説明文にポケモン名が含まれると名前推測のヒントになるため、自然な代名詞に置換する。

- 英語：文脈に応じて "this Pokémon"（単数）/ "of these Pokémon"（複数）
  - 複数形の判定：直前の単語が several / many / few / multiple 等 → 複数形
  - 文頭 → 先頭大文字 "This Pokémon"
- 日本語："この ポケモン"

AI スコアリングには元の（マスクなし）テキストを使い、採点精度に影響しない。捕獲結果とコレクションでは元の名前を表示する。

### 実装への影響

`QuestSession` から `NameMultiplier` / `WrongGuesses` を削除し、`BaseStatTotal` / `BallType` に置き換える。BST は PokeAPI `/api/v2/pokemon/{id}` の `stats` フィールドから自動計算する。
