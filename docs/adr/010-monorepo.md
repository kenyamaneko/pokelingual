# ADR-010: モノリポ構成を採用する

## ステータス

Accepted

## 結論

backend / frontend / shared を単一リポジトリで管理するモノリポ構成にする。

## 背景・課題

backend / frontend をともに TypeScript にした狙いは、API 契約の型をコードとして共有できるようにすることにある（ADR-001）。リポジトリを分けると、共有する型パッケージ（`shared/api-types`）の変更をバージョニングして両リポジトリに配信する手間が生じ、型共有のメリットを活かしにくい。モノリポであれば同一コミットで backend / frontend / 共有型を一括して変更でき、型のズレをビルド時に検出できる。
