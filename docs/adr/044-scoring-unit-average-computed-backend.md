# ADR-044: 採点単位の平均計算をbackend側で行う

## ステータス

Accepted

## 結論

意味単位分解方式の翻訳採点 (BDR-012) で、LLMには意味単位ごとの判定値のみを配列で出力させ (単位の説明文は出力させない)、素点への平均計算はbackend側の純粋関数で行う。

## 背景・課題

意味単位分解方式の導入にあたり、LLMの出力形式には複数の選択肢があった。単位ごとの説明文まで含めた内訳を出力させる案は、Gemini 2.5 Flash の出力課金 (入力の8倍以上) の影響で1回あたりのコストが現状比で大きく増える見込みだった。一方で、LLMに最終スコアの計算まで行わせて素点のみを返させる案は出力コストを抑えられるが、LLMが実際に単位ごとに独立して判定し平均したのか、従来通り1つの数字を勘で出しているだけなのかを検証できない。

## 不採用案

- **LLMに単位の内訳 (説明文+判定値) まで出力させる案**: 出力トークンが大きく増え、コスト増が採点1回あたりの許容範囲を超えるため不採用。
- **LLMに最終スコアの計算まで行わせ、backendは受け取るだけにする案**: 出力コストは現状に近い水準を保てるが、平均計算の正しさを検証・テストできず、単位分解を経ずに holistic な値へ回帰しても気づけないため不採用。
