# ADR-042: 覚える技ヒントの選出をクエスト開始時に行う

## ステータス

Accepted

## 結論

スナップショットには、各ポケモンがレベルアップで覚えうる技の候補を全件、日本語名として保存する。ヒントとして見せる最大3件の選出は、生成時ではなくクエスト開始時にbackend serviceが乱数で行う。ADR-040を置き換える。

## 背景・課題

ADR-040は技ヒントを生成時に3つへ確定する設計を採ったが、この設計では同じ種に何度出会っても常に同じ3技しか開示されず、繰り返しの遭遇に新しい発見が無い。

QuestServiceは出題ポケモンの抽選・幻/伝説抽選・複数バージョンがある説明文の選択を、クエスト開始時に注入済みの乱数源 (RandomSource) で既に行っている。技ヒントの選出もこれと同じ箇所・同じ乱数源で行えば、抽選ロジックの新たな置き場所を用意する必要はない。ADR-040が不採用理由に挙げた「抽選ロジックをbackend serviceに新たに持ち込む必要がある」というコストは、この既存の仕組みに1箇所加えるだけで済むため、当初の見積もりより小さい。スナップショットには全候補を保存するだけで済み、実行時にPokeAPIを参照しない設計 (ADR-031) は変わらず維持される。

## 不採用案

ADR-040の生成時確定案を維持する案。実装は単純だが、同じ種であれば毎回同じ3技しか開示されず、繰り返し遭遇時の体験に乏しい。この制約に対して、実行時抽選による実装コストの増分 (既存の乱数源を1箇所で呼ぶだけ) は小さく、見合わないため不採用とした。
