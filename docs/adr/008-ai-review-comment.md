# ADR-008: 翻訳スコアリングに AI レビューを追加

## ステータス

Accepted

## 結論

点数の理由が分かるフィードバックを与えるため、Gemini API のプロンプトに `review` フィールドを追加し、日本語レビューを返す。スコアラベル（「効果は ばつぐんだ！」等）と区別するため、AI フィードバックは `review` に統一する。ユーザーは具体的な講評と難しい単語の解説を得られ、命名の分離でコードの可読性も上がる。

## 背景・課題

翻訳スコアリングでは 0-100 の数値スコアのみを返していた。ユーザーが「なぜこの点数なのか」を理解する手がかりがなく、学習体験としてフィードバックが不十分だった。また、スコアラベルと AI フィードバックの両方を「コメント」と呼ぶと紛らわしかった。

## 詳細

### 変更箇所

- **ScoreResult**（`model/quest.go`）：`Review string` フィールド追加
- **ScoreResponse**（`service/quest_service.go`）：`Review string` フィールド追加、サービス層でパススルー
- **GeminiService**（`service/gemini_service.go`）：プロンプトに review 要求を追加
- **devmock**：スコア範囲に応じた固定レビューを返す
- **フロントエンド**：`QuestPage` で「きみの 翻訳」の下にレビューを表示

### レビューのガイドライン

- 日本語で 2-3 文（100 文字以内）
- PokeAPI の日本語説明文スタイル（簡単な漢字 + スペース区切り）
- 適宜含む内容：翻訳の良い点（語彙選択、文法など）、改善点（ニュアンスの欠落、不自然な表現、誤訳）、難しい英単語・フレーズの解説
- 励ましつつ正直に、ポケモン博士のような口調で
- 例：`「friskily」は 元気よく 跳ね回る という 意味だよ。前半は よく 訳せているが 後半の ニュアンスが 少し 惜しい。`

### 命名規則

| 概念 | コード上の名前 | 表示場所 |
|------|---------------|---------|
| スコアラベル（「効果は ばつぐんだ！」等） | `getScoreLabel()` | ScoreDisplay |
| AI フィードバック | `review` | QuestPage（「きみの 翻訳」の下） |
