# ADR-008: 翻訳スコアリングに AI レビューコメントを追加

## ステータス

採用済み

## コンテキスト

翻訳スコアリングでは 0-100 の数値スコアのみを返していた。
ユーザーが「なぜこの点数なのか」を理解するための手がかりがなく、学習体験としてフィードバックが不十分だった。

## 決定

Gemini API のプロンプトに `comment` フィールドを追加し、一行の日本語レビューコメントを返すようにする。

### 変更箇所

- **ScoreResult**（`model/quest.go`）: `Comment string` フィールド追加
- **ScoreResponse**（`service/quest_service.go`）: `Comment string` フィールド追加、サービス層でパススルー
- **GeminiService**（`service/gemini_service.go`）: プロンプトに comment 要求を追加
- **devmock**: スコア範囲に応じた固定コメントを返す
- **フロントエンド**: `ScoreDisplay` コンポーネントでコメントを表示

### コメントのガイドライン

- 日本語の短い一文（30文字以内）
- PokeAPI の日本語説明文スタイル（簡単な漢字 + スペース区切り）
- 例: `意味は 合っているが 表現が 少し 不自然だ`

## 結果

- ユーザーは点数の理由を一目で理解できるようになった
- Gemini API へのリクエスト数は変わらない（既存のスコアリングリクエストにコメントを追加しただけ）
- devmock でもスコア範囲に応じたコメントを返すため、ローカル開発でも体験を確認可能
