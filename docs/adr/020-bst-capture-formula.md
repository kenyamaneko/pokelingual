# ADR-020: 種族値ベースの捕獲率計算式

## ステータス

Accepted

## 結論

強いポケモンほど捕まえにくくしてゲーム性を高めるため、捕獲率を種族値（BST）とスコアを説明変数とするシグモイド関数で計算し、名前当ての結果で決まるボール倍率を掛ける。出題文中のポケモン名は伏せ字にし、名前推測のヒントにならないようにする。強敵の捕獲に達成感が生まれ、名前推測がより挑戦的になる。

## 背景・課題

従来の捕獲率は score × nameMultiplier × guessPenalty の単純な線形計算で、全ポケモンが同じ条件で捕まった。強いポケモン（高種族値）を捕まえた達成感がなく、ゲーム性が物足りなかった。ポケモン本家では種族値が高いほど捕獲が難しく、同様の体験を翻訳ゲームでも実現したかった。

具体的な式の係数は体感値をもとにしたチューニング値であり、固定値としては本 ADR に書かない（Amendment で複数回調整している）。

## Amendment: 2026-02-14 捕獲率式の係数を再調整

捕獲率式の係数を調整した。狙いはスコア 90 + スーパーボールなら伝説ポケモン（BST 680）でもほぼ確実に捕獲できるようにすること。ボール倍率（モンスターボール 1.0x / スーパーボール 2.0x / ハイパーボール 3.0x）は変更しない。

## Amendment: 2026-07-16 ボール補正を倍率から加算方式に変更

ボール種別ごとの捕獲確率補正を、基礎確率への倍率からロジットへの加算ボーナス（モンスターボール 0 / スーパーボール +1.5 / ハイパーボール +3.0）に変更した。倍率は廃止する。基礎確率が低い場面（ノーダメージなど）では倍率による底上げが小さく、種族値の高いポケモンがどのボールでもほぼ捕まえられなかった。弱らせていなくても良いボールを使えば多少は捕まえられるという本家の体感に合わせるため、ロジットへの加算に変更した。

シグモイド関数の出力は加算後も 1.0 を超えないため、旧実装にあった確率の上限クランプは不要になった。この変更により、上記 2026-02-14 の Amendment にある「スコア 90 + スーパーボールなら BST 680 でもほぼ確実」という前提は成り立たなくなる（同条件で約 94%）。ハイパーボールであれば、最終評価点の上限 (99) で BST 680 でも 99% を超え、同等の体感を保っている。

## Amendment: 2026-07-18 マスターボールは本計算式をバイパスする

伝説・幻ポケモンの確定捕獲 (BDR-011) で確定するマスターボールは、本 ADR の計算式を使わない。捕獲確率は種族値合計によらず一律 1.0 とし、抽選も行わず必ず捕獲に成功する。レアな伝説・幻を高スコアで仕留めても確率で逃す落胆を避けるための例外であり、種族値ベースの難度設計自体は他のボール種別で維持する。

## Amendment: 2026-07-18 ボール補正を運用時に設定可能にする

ボール補正 (0 / +1.5 / +3.0) を env 経由で運用時に変更できる設定値にした。既定値は変えていない。

## Amendment: 2026-07-19 捕獲率式の係数を再調整(スコアの影響を強化)

捕獲率式の係数を調整した。中程度のスコアでも十分捕獲できてしまい、翻訳の精度を上げる動機が薄れていたため、低〜中スコア帯の捕獲率を種族値帯を問わず引き下げ、高スコア帯の捕獲率は現行に近い水準を保つようにした。種族値による難度の差は現行に近い比率を保ちつつ、スコアによる差をより大きく開くことを優先している。
