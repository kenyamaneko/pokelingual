# ADR-001: バックエンド/フロントエンドをともに TypeScript にする

## ステータス

Accepted

## 結論

フロントエンドは採用フレームワークによらず TypeScript 前提になる。バックエンドもこれに合わせて TypeScript を採用し、API 契約の型をコードとして共有できるようにする。

## 背景・課題

バックエンドの実装言語には TypeScript 以外に、開発者が実務で使える Go・C# も選択肢にあった。フロントエンドが TypeScript である以上、バックエンドも揃えれば API 契約の型をフロント/バックで共有でき、二重定義によるズレを防げる。

## 不採用案

- **Go / C# をバックエンドに採用する**：言語自体に不足はないが、フロントエンドと型を共有できないため、API 契約をドキュメントや手動同期で保つ必要が生じ、二重定義のズレを防げない。
