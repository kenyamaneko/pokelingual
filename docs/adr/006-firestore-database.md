# ADR-006: DBに Firestore を採用する

## ステータス

Accepted

## 結論

データストアには Firestore を採用する。

## 背景・課題

開発者が慣れていて手軽なことに加え、Firebase Auth・Cloud Run など他の Google Cloud サービスとの相性がいい。Cloud SQL も検討したが、コストを抑えたい個人開発でリレーショナルDBの常時起動コストを払う理由がなく、想定するデータモデルもそこまで複雑にならない見込みだったため、リレーショナルな表現力を妥協して Firestore を選んだ。

## 不採用案

- **Cloud SQL**：リレーショナルな表現力は得られるが、常時起動インスタンスのコストがかかる。データモデルがそこまで複雑にならない想定の下では、コストに見合わない。
