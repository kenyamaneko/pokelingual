# ADR-039: テスト観点カタログのトップレベル describe を `[タグ]` で上位グルーピングする

## ステータス

Accepted

## 結論

トップレベル describe 名の先頭に `[認証系]` のような `[タグ]` を付けられるようにする。[テスト観点カタログ](024-behavior-catalog-from-test-names.md)の生成時にタグを上位グループ見出しとして抽出し、タグを剥がした describe 名をその見出しの下に並べる。同じタグは、複数のテストファイルに分かれていても 1 グループに集約する。タグの無いトップレベル describe は「その他」にまとめる。目印をテストコード側（describe 名）に置くことで、describe のリネームやファイル移動と一緒に動き、別ファイルの保守を要らなくする。

## 背景・課題

テスト観点カタログはセクション内に多数の describe がフラットに並ぶだけで、関連するテストがひとまとまりに見えなかった。グルーピングの目印をどこに置くかが論点になった。

## 不採用案

- **外部設定ファイルでグルーピングを指定する**：セクション → グループ → describe 名の対応と順序を別ファイルに書き、スクリプトが読んでグルーピングする案。テスト名を変えず順序も完全に制御できるが、describe をリネームすると設定がズレる（CI で検知はできるが別ファイルの保守が要る）。取りこぼしを防ぐには全 describe を設定ファイルへハードエラーで強制列挙する必要もある。タグ案はズレず・別ファイル不要・段階導入できるため、こちらを不採用とした。

## Amendment: 2026-07-18 タグの無いセクションでは「その他」でまとめない

セクション内にタグ付き describe が 1 件も無いときは「その他」でまとめない。まとめる相手がいないセクションまで機械的に 1 段深くすると、見出しが箇条書きに格下げされるだけで読みやすさの向上が無いため。
