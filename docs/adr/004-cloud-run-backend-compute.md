# ADR-004: バックエンドの実行基盤に Cloud Run を採用する

## ステータス

Accepted

## 結論

バックエンドは Cloud Run（サービス）で実行する。

## 背景・課題

バックエンドは複数エンドポイントを持つ1つのサービスであり、単一目的の関数を前提とする Cloud Run Functions には載せきれない。一方でリクエストの少ない個人開発の負荷特性上ステートレスかつコールドスタート許容で問題なく、常時起動させておく Compute Engine を立てるコストは見合わない。

## 不採用案

- **Cloud Run Functions**：複数エンドポイントを持つバックエンドを単一目的の関数の集まりとして分割する必要があり、構成が煩雑になる。
- **Compute Engine**：ステートレスでコールドスタートを許容できる負荷特性に対して、常時起動のインスタンス費用を払う理由がない。
