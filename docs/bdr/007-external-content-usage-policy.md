# BDR-007: 外部コンテンツの利用方針

## ステータス

Accepted

## 結論

PokeLingual が扱うポケモンのデータと画像は、いずれもポケモン社が著作権を持つ。次のとおり扱う。

- **フェアユースの遵守**：PokeAPI から取得したリソースはキャッシュし、実行時に繰り返し取得しない
- **再配布の禁止**：ポケモン社の著作物を、公開リポジトリなど誰でも取得できる場所に複製して置かない
- **複製の最小化**：キャッシュは非公開の範囲に留める。複製せずに済むものは複製しない

アプリの機能として画面に表示することと、データセットとして第三者が取得できる形で再配布することを区別する。前者は続け、後者は行わない。

## 背景・課題

### PokeAPI のフェアユース

PokeAPI は無償で公開されている教育目的のサービスで、フェアユースポリシーに「取得したリソースは必ずローカルキャッシュすること」と明記し、従わない利用者は IP アドレスを恒久的に締め出すと宣言している。2018 年 11 月の静的ホスティング移行でレート制限自体は撤廃されたが、リクエスト頻度を抑える責任は利用者に残る。

PokeLingual はポケモンの名前・説明文・タイプ・種族値を PokeAPI に全面的に依存している。キャッシュを持つことは選択肢ではなく要求である。

### 著作物としてのポケモンデータ

図鑑の説明文はゲーム中の創作的表現であり、ポケモン社の著作物にあたる。ポケモンの名前も同様で、商標でもある。公式アートワークとアイテムのスプライトも当然に著作物である。画像とテキストのあいだに、著作権上の扱いの違いはない。

参照元リポジトリのライセンスは、この判断を変えない。`PokeAPI/sprites` は CC0 1.0 だが、README に「All image contents within are Copyright The Pokémon Company」と明記されている。`PokeAPI/api-data` は BSD-3-Clause である。いずれもリポジトリの構成物に対するライセンスであって、収録されたポケモン社の著作物の権利を移転しない。

ファンプロジェクトがスプライトやデータを自前ホストする例は多く、PokeAPI 自身がそれにあたる。しかし慣行の存在は適法性の根拠にならない。PokeLingual は収益化していない個人開発のファンアプリであり、避けられる係争リスクを負う理由がない。

## 詳細

公開リポジトリにポケモン社の著作物を含めない。チュートリアル用の説明文（`tutorialFixtures.ts`）はゲームの図鑑テキストではなく独自に書き起こしたもので、この状態を維持する。

自前ホストや再配布が必要になったときは、この BDR を置き換える新しい BDR で判断する。実装の都合で個別に例外を作らない。
