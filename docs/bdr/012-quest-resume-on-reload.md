# BDR-012: クエストのリロード再開

## ステータス

Accepted

## 結論

進行中のクエストセッションがある間は、画面をリロードしても同じポケモン・同じ段階から再開する。対象は出題(翻訳)・採点後(名前当て)・捕獲待機の3段階。エラー画面の「もう一度探す」も、リロードと同じ判定で生きたセッションがあれば再開する。

## 背景・課題

出題画面をリロードすると必ず新しいポケモンが抽選し直され、狙ったポケモンが出るまでリロードを繰り返す厳選ができてしまっていた。また、採点後や名前当て中に一時的な通信エラーで画面が失われると、セッション自体は生きているのに進行を再開できなかった。

## 詳細

- 対象外は次の2つ。場所選択画面(セッション開始前)のリロードは、場所の再抽選を許容する。チュートリアルはリロードで最初から再開する。
- 捕獲後(演出・結果画面)は、捕獲した時点でセッションが失効するため再開の対象にならない。
- 名前当ての正誤バナーのような一過性の演出は再現しない。ヒントで開示済みの情報(タイプ・技)は再表示する。
- 名前当てが確定した状態(正解・スキップいずれも)でのリロードは、捕獲待機の段階として再開する。
