# BDR-004: 翻訳結果画面のリビール順序とアニメーション演出

## ステータス

Accepted

## 結論

翻訳結果画面 (`guessing` フェーズ) は、採点結果を一括表示せず、ダメージメーターのアニメーション → こうか表示（「効果は バツグンだ！」等）→ 博士のコメントの順に段階的に開示する。ダメージメーターは満タンからスコアに応じた残量まで動的に減少させ、博士のコメントはメーターのアニメーション完了後に1文字ずつ送り出すタイプライター演出で表示する。アニメーション実装にはライブラリの利用を許可する。

## 背景・課題

現状は採点結果（スコア・こうか表示・HPバー・博士のコメント）が採点完了と同時に静的に一括表示されており、評価がどう出るかの緊張感がない。結果を段階的に開示することで、評価点が確定するまでのドキドキ感をユーザーに与え、ゲームとしての没入感を高める。
