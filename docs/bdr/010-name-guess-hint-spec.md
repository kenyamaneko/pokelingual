# BDR-010: 名前当てのヒント仕様

## ステータス

Accepted

## 結論

名前当てにヒントを追加する。ヒントを要求すると出題ポケモンのタイプを表示し、代償として名前当ての挑戦回数を1回消費する。挑戦回数の消費は以下のルールに従う。

- ヒントは残り挑戦回数が2回以上のときのみ要求できる。消費後に少なくとも1回は名前当てへの挑戦が残るようにするため。
- ヒントは1クエストにつき1回だけ要求できる。返す情報 (タイプ) は常に同じで、複数回要求しても新しい情報が増えないため。
- 残り挑戦回数が1回になり、かつヒントを一度も使っていないときは、ヒントがもう要求できない旨を案内する。無駄な挑戦回数の消費を避けるため、ボタンを消すだけでなく理由を伝える。ヒントを既に使用済みのときはタイプが既に画面に出ているため、この案内は不要。
- 初回チュートリアルの名前当て画面にはヒントを出さない。チュートリアルは案内文で正解の名前を提示しており、ヒントを見る意味がないため。

技をヒントに使う拡張は対象外 (別issueで管理)。

## 背景・課題

名前当ては最大3回まで挑戦できるが、ノーヒントで英語名を当てるのが難しいケースがあり、回答権を対価にヒントを見られる機能が要望された (Issue #176)。技はスナップショットに含まれないため追加取得コストがかかり、タイプは出題時点で取得済みのため追加コストが無い。まずタイプ表示のみをMVPとする。

ヒントは「挑戦」ではなく「情報開示」だが、同じ回数カウンタを消費する設計にしたため、残り回数が少ない状況での扱いが未定義だった。ヒントで最後の挑戦機会まで消費してしまうと、情報を得ても実際に答える機会がないまま終わる矛盾が生じる。この矛盾を避けるため、消費後も必ず1回は挑戦が残る条件 (残り2回以上) を設けた。
