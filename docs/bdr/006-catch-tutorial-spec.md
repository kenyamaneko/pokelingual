# BDR-006: 初回チュートリアルの仕様

## ステータス

Accepted

## 結論

メニューの「ポケモンを探す」を実行する時点でチュートリアル未完了なら、本番の判定 (LLM 採点・確率的な捕獲成否) の代わりに、操作を教えることに特化した固定シナリオのチュートリアルを実行する。登場ポケモンはピカチュウ固定。英文当て・ポケモン名当てはモーダルで入力すべき文字列をそのまま案内し (正答を推測させない)、採点は必ず100点、捕獲は必ず成功として結果を見せる。

チュートリアルを一度完了すると以降は自動的に表示されなくなるが、メニューの「ポケモンを探す」の下にボタンではない導線を常設し、完了後もいつでも手動で再視聴できる。

## 背景・課題

新規ユーザーが初めて「ポケモンを探す」を実行したとき、英文当て・ポケモン名当て・ボール捕獲という一連の操作を本番の難易度のまま体験させると、操作方法を覚える前に失敗体験をしてしまう。操作の流れそのものを、正答を教えながら1回だけ確実に成功する形で体験させ、以降の本番プレイへの心理的ハードルを下げる。

## 詳細

- **発火条件**：「ポケモンを探す」実行時にチュートリアル未完了なら、本番のクエストの代わりにチュートリアルを開始する。
- **再視聴導線**：メニュー画面の「ポケモンを探す」の下に、ボタンではないテキストリンク形式の導線を常設する。完了状態に関わらず、ここからは何度でもチュートリアルを開始できる。
- **登場ポケモン**：ピカチュウ (図鑑番号25) 固定。
- **英文当てステップ**：
  - チュートリアル専用の固定英文 (例："It is an Electric-type Mouse Pokémon.") を表示する。
  - モーダルで「電気タイプのねずみポケモン、と入力してください」のように、入力すべき文字列をそのまま案内する。
  - 入力に「電気」「でんき」のいずれか、かつ「ネズミ」「ねずみ」のいずれかが含まれていなければ送信を拒否する。
  - 両方の条件を満たしたときのみ送信を許可し、採点結果は必ず100点として表示する。
- **ポケモン名当てステップ**：
  - モーダルで「ピカチュウ または pikachu と入力してください」と案内する。
  - 入力値 (前後空白除去) が「ピカチュウ」または「pikachu」(大文字小文字を区別しない) のいずれとも完全一致しなければ送信を拒否する。
  - 一致したら捕獲ステップに進む。本番と異なりボール種別による分岐は行わない。
- **捕獲ステップ**：ボール使用演出後、必ず捕獲成功の結果を表示する。
- **完了の記録**：捕獲成功の結果画面に到達した時点でチュートリアル完了として記録する (永続化方式は ADR-030)。

## Amendment: 2026-07-14 案内の表示形式変更

英文当て・ポケモン名当ての案内を、背景を操作できない全画面モーダルから、入力欄の直上に常時表示する吹き出しに変更した。モーダルは閉じないと入力できず、閉じた後は何を入力すべきだったかを見返す手段がなかった。吹き出しはステップ開始から一定の遅延を置いて表示され、以後は閉じる操作なくステップ終了まで表示され続ける。案内する文言・入力の検証ルールは変わらない。
