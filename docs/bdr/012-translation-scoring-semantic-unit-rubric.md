# BDR-012: 翻訳採点ルーブリック (意味単位分解方式)

## ステータス

Accepted

## 結論

翻訳採点は、原文を意味単位 (節・重要語彙・構文など、長さによらず訳出の正確さを独立に判定できる最小単位) に分解し、単位ごとに次の6段階のいずれかを判定値として割り当てる。

- 1.0: 正しく訳せている
- 0.8: 全体的に正しいが、一部惜しいミスがある
- 0.6: 文意は拾えているが、訳が不自然
- 0.4: 文意を理解できているか怪しいが、単語は部分的にあっている
- 0.2: 一部の単語の意味は答えられているが、文章は破綻している
- 0.0: 完全な誤訳、または脱落

判定値が0.8以下の単位でも、特に秀逸な訳であれば0.1を加算する。素点 (0-100) は「判定値の合計 ÷ 単位数 × 100」(四捨五入) とし、単位の個数の多寡ではなく原文に対する到達率で採点する。この素点は既存の最終評価点変換 (BDR-009) にそのまま入力する。

## 背景・課題

従来のプロンプトは「90-100点: 意味が正確」のような5段階の帯域指示のみで、帯域内のどの数値を選ぶかの粒度指示が無かった。LLMは帯域の中心や丸めた値に寄りがちで、結果として最終評価点変換後の値 (画面表示) も特定の値に偏って出ることが多かった。

原文を1つの塊として評価させると、LLMは holistic に1つの数字を勘で決めやすい。意味単位に分解して単位ごとに判定させることで、原文の長さや複雑さに応じて判定の粒度そのものが上がり、値の集中を緩和できる。

## 詳細

- 未訳・省略は「完全な誤訳、または脱落」として当該単位を0.0と判定する。講評の文面では、訳せなかった箇所を不注意と決めつけず優しく導く既存の講評文面方針と両立させる。スコアには反映するが、講評のトーンは別扱いとする。
- 単位の分解・判定はLLMが行い、素点への平均計算はbackend側で行う (設計判断はADR-044)。
