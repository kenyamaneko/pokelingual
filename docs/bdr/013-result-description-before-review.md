# BDR-013: 翻訳結果画面の解答例と博士のコメントの開示順序

## ステータス

Accepted

## 結論

翻訳結果画面 (`guessing` フェーズ) は、解答例 (日本語版の図鑑の説明) → 博士のコメント → HP 減少 → ダメージ数値の順に段階的に開示する。解答例と博士のコメントはタイプライター演出で1文字ずつ表示し、その開示が終わってから HP バーと HP 数値を満タンから残量まで連動して減少させ、最後にダメージ数値とこうか表示を出す。HP とダメージの数値はどちらも分母 (100) を表示しないパーセント表記とする (例: 70/100 → 70%)。

## 背景・課題

BDR-008 は博士のコメント → 解答例の順で開示していた。画面レイアウトでは出題英文・君の翻訳・解答例が同じカードにまとまり、博士のコメントは別カードに分かれて配置されている。この配置のまま博士のコメントを先に開示すると、原文と日本語の解答例の表示が時間的に分断され、両者の対応関係が読み取りづらくなっていた。解答例を先に開示することで、原文・訳文・解答例のまとまりを崩さずに結果を確認できるようにする。
