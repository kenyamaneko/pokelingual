# BDR-009: 翻訳採点の最終評価点変換と "こうか" 表示規則

## ステータス

Accepted

## 結論

LLM が返す素点 x (0-100) を、最終評価点 y = round((x - 10) * 1.1) (ただし (x - 10) * 1.1 <= 0 のときは 0) に変換する。以降のスコア (`ScoreResponse.score` / 図鑑の `best_score` / 捕獲確率計算) はすべて y を用いる。「効果が ないみたいだ...」は y が 0 のとき (素点 10 以下) にのみ表示し、一撃必殺 (「いちげきひっさつ！」) は廃止して最上位帯も「効果は バツグンだ！」とする。

## 背景・課題

「効果が ないみたいだ...」は従来、素点が 0 点のときにしか出ずレアすぎた。ハズレ方が大きいときに出るようにしたい。また最終評価が 100 点 (満点) に達すると、ポケモンの世界観 (HP を削り切る＝瀕死にすると捕まえられない) と矛盾する。最終評価点の上限を 99 に抑えることで、この世界観に合わせる。

## 詳細

- 変換は backend の採点確定時に一度だけ行い、frontend では再変換しない (二重適用の防止)。
- 丸めは四捨五入。x は 0-100 の整数のため、y は 0-99 の整数になる (x<=10 で 0、x=11 で 1、x=100 で 99)。
- この y 圧縮に合わせた捕獲確率計算 (ADR-020) の再調整は行わない。捕獲がやや難化することは許容する。

## Amendment: 2026-07-20 素点 x の算出元を明記

BDR-012 (意味単位分解方式) の導入により、素点 x は LLM が直接返す値ではなく、LLM が返す意味単位ごとの判定値配列から backend が算出する値になった。変換式・丸め・値域はいずれも変わらない。
